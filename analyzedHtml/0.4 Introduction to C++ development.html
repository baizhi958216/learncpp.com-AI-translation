在我们能够编写和执行第一个C++程序之前，我们需要更详细地了解C++程序是如何开发的。这里有一个图示概述了一个简单的步骤：

步骤1：定义您想要解决的问题

这是“什么”步骤，您在此确定您要解决的问题。产生您想要编程的初始想法可能是最容易的步骤，或者是最难的步骤。但概念上，这是最简单的。您只需要一个可以明确定义的想法，您就可以准备好进入下一步了。

以下是一些例子：

  • “我想写一个程序，允许我输入许多数字，然后计算平均值。”
  • “我想写一个程序，生成一个2D迷宫并让使用者在其中导航。如果用户到达终点，他们就赢了。”
  • “我想写一个程序，读取一个股票价格的文件，并预测股票是会上涨还是下跌。”

步骤2：确定您将如何解决问题

这是“如何”步骤，您在此确定您将如何解决步骤1中提出的问题。这也是软件开发中最常被忽视的步骤。问题的核心是有很多方法可以解决问题——然而，这些解决方案中有些是好的，有些是坏的。程序员经常有一个想法，坐下后立即开始编写解决方案。这通常会产生一个属于坏类的解决方案。

通常，好的解决方案具有以下特点：

  • 它们是直接的（不复杂或令人困惑）。
  • 它们有良好的文档记录（尤其是在任何假设或限制方面）。
  • 它们是模块化的，因此以后可以重新使用或更改某些部分而不影响程序的其他部分。
  • 当发生意外情况时，它们可以优雅地恢复或提供有用的错误信息。

当您坐下来立即开始编码时，您通常会想“我想做<某事>”，所以您实现能最快达到目的的解决方案。这可能导致程序脆弱、难以以后更改或扩展，或者有很多错误。一个bug是指任何使程序无法正确运行的编程错误。

顺便说一下…

“bug”这个词最早是由托马斯·爱迪生在19世纪70年代使用的！不过，这个词在1940年代因工程师发现一只真正的飞蛾卡在早期计算机的硬件中导致短路而广为流传。报告该错误的日志本和飞蛾现在都是美国历史博物馆的一部分。它可以在 这里查看。

各种研究表明，在复杂的软件系统中，程序员只有10%到40%的时间用于编写初始程序。其余的60%到90%的时间用于维护，这可能包括 调试（去除错误）、更新以应对环境变化（例如在新操作系统版本上运行）、增强（微小的更改以提高可用性或功能），或内部改进（以提高可靠性和可维护性）1.

因此，花一点额外的时间在一开始就思考解决问题的最佳方法、您所做的假设以及如何为未来做计划，是值得的，这样可以节省您以后很多时间和麻烦。

我们将在以后的课程中更多地讨论如何有效地设计解决问题的方法。

步骤3：编写程序

要编写程序，我们需要两件事：首先，我们需要对编程语言的知识——这就是这些教程的目的！其次，我们需要一个文本编辑器来编写和保存我们的C++程序。我们输入到文本编辑器中的C++指令称为程序的 源代码（通常简称为代码）。您可以使用任何文本编辑器编写程序，甚至像Windows的记事本或Unix的vi或pico这样的简单工具。

在基本文本编辑器中输入的程序看起来如下所示：

#include <iostream>

int main()
{
    std::cout << "Here is some text.";
    return 0;
}

然而，我们强烈建议您使用专为编程设计的编辑器（称为代码编辑器）。不用担心您目前还没有。我们将很快介绍如何安装一个代码编辑器。

一个典型的编程用编辑器有几个功能，使编程更容易，包括：

  1. 行号。当编译器给出错误时，行号很有用，因为典型的编译器错误会指出： 一些错误代码/消息，第64行。如果没有显示行号的编辑器，找到第64行可能会很麻烦。
  2. 语法高亮和着色。语法高亮和着色会改变程序的不同部分的颜色，使您更容易识别程序的不同组件。
  3. 明确的、等宽字体（通常称为“等宽字体”）。非编程字体往往很难区分数字0和字母O，或者区分数字1、小写字母l和大写字母I。一个好的编程字体会确保这些符号在视觉上有所区别，以确保不会误用其中一个代替另一个。所有代码编辑器都应该默认启用这个功能，但标准文本编辑器可能没有。使用等宽字体（所有符号宽度相同）使正确格式化和对齐代码变得更容易。

这是一个带有行号、语法高亮和等宽字体的C++程序示例： 

#include <iostream>

int main()
{
    std::cout << "Here is some text.";
    return 0;
}

注意这比未高亮的版本要容易得多。我们在本教程中展示的源代码将包含行号和语法高亮，以便于理解。

提示

Coding Font 和 Programming Fonts 都有很棒的工具，可以让您比较不同的编程字体，看看哪些最适合您。

对于高级读者

因为源代码使用ASCII字符编写，编程语言使用一定量的ASCII艺术来表示数学概念。例如，≠ 不属于ASCII字符集，因此编程语言通常使用 != 来表示数学不等式。

一些编程字体，如 Fira Code，使用连字来将这些“艺术”重新组合成一个字符。例如，Fira Code 会显示 ≠（使用与双字符版本相同的宽度）而不是 !=。有些人觉得这更容易阅读，其他人则更喜欢保留原始字符的字面解释。

许多简单的C++程序只有一个源代码文件，但复杂的C++程序可以有数百甚至数千个源代码文件。

您程序中的每个源代码文件都需要保存到磁盘上，这意味着每个源代码文件都需要一个文件名。C++ 对命名文件没有任何要求。但是，事实上的标准是将为程序创建的第一个/主要源文件命名为 main.cpp。文件名（main）使得很容易确定哪个是主要的源代码文件，.cpp 扩展名表明该文件是一个 C++ 源代码文件。

您偶尔可能会看到第一个/主要的源代码文件以程序名称命名（例如 calculator.cpp, poker.cpp）。您也可能会偶尔看到其他扩展名（例如 .cc 或 .cxx）。

最佳实践

将每个程序中的第一个/主要源代码文件命名为 main.cpp。这有助于确定哪个源代码文件是主要的。
